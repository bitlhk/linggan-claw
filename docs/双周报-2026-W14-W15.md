## 🏗️ 二、技术基础设施

### 新部署的服务

为了支撑"灵枢 · 深度求索"，新引入并跑通了团队自研的两个开源组件：

| 服务 | 作用 | 端口 | 状态 |
|---|---|---|---|
| **hi-agent** | TRACE 五阶段任务执行框架 | 8080 | online ✅ |
| **agent-kernel** | 企业级 Agent 运行时内核 | 8400 | online ✅ |
| **OpenClaw Gateway** | Agent 工具调度网关 | 18789 | online ✅ |
| **灵虾后端** | 主服务 | 5180 | online ✅ |

### 协作架构梳理

正式确立了灵虾的多 Agent 协作模型：

```
DB 持久化（任务状态/审计） + 内存流缓存（实时打字效果） + Gateway 调用（实际执行）
```

三件套混合架构，既有完整审计链路又有毫秒级实时性。已写成专题文档。
