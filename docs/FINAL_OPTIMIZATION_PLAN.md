## 🟡 需要验证的优化

### 字体文件优化（最大潜力）

**当前状态**：
- **字体文件**: 784 个，22.6MB
- **CSS 引用**: 多个字符子集（100-119等）
- **字重**: 400, 500, 600, 700

**问题**：
- 可能包含多个字符子集，但我们可能只需要常用字符
- 字体文件已打包到 CSS，但浏览器仍需要独立的 woff 文件

**优化方案**：
1. **分析实际使用的字符**：检查代码中使用的字符范围
2. **使用字体子集**：只包含需要的字符集
3. **删除未使用的字体文件**：只保留 CSS 中引用的

**预期收益**: 节省 **15-20MB**

**实施难度**: 🟡 中等（需要测试确保字体正常）

**建议**: 先实施简单优化，字体优化作为后续优化
