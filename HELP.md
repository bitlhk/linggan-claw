# 员工智能体使用指南

员工智能体用于把一个可长期工作的 AI 助手交给每个用户或团队。它可以聊天、处理文件、安装技能、记住偏好、绑定微信/飞书、创建定时任务，也可以和组织内其他智能体协作完成工作。

这份文档按“支持哪些场景、怎么使用”来写。部署和排障放在最后，普通用户可以先看前两节。

---

## 1. 快速开始

### 申请一个员工智能体

1. 登录平台。
2. 在首页点击“申请员工智能体”。
3. 等待创建完成后进入工作台。
4. 在聊天框里直接输入你的任务。

每个员工智能体都是独立实例。它的对话、记忆、技能、文件、定时任务和渠道绑定会彼此隔离，不会读取其他用户或其他智能体的数据。

### 工作台里有哪些入口

| 入口 | 用来做什么 |
|---|---|
| 聊天 | 问答、写作、分析、文件处理、生成结果 |
| 技能 | 安装、上传、管理当前智能体可用的能力 |
| 技能广场 | 从平台或组织共享能力中安装技能 |
| 定时任务 | 创建每天、每周、单次或 cron 任务 |
| 频道 | 绑定微信、飞书等外部消息渠道 |
| 记忆 | 管理智能体长期记住的偏好和背景 |
| 工作空间 | 查看、上传、下载和删除文件 |
| 协作 | 发起或处理组织协作任务 |
| 设置 | 调整主题、模型、显示偏好等 |
| 文档 | 查看当前帮助文档 |

## 2. 常用场景怎么用

### 场景一：日常问答和写作

适合用来写邮件、总结材料、解释概念、整理会议纪要、起草方案。

可以这样说：

```text
帮我把这段会议记录整理成一页行动清单，按负责人、截止时间、风险点分组。
```

```text
用更正式的商务语气改写这封客户邮件，保留核心诉求，不要太强硬。
```

使用建议：

- 把背景、目标、输出格式一起说清楚。
- 如果要给领导或客户看，明确“语气正式、结论先行、不要太口语”。
- 如果结果不够贴合，可以继续追问“压缩到 300 字”“改成表格”“更像汇报口径”。

### 场景二：读取和处理文件

适合用来总结文档、分析表格、生成提纲、检查内容差异。

使用方式：

1. 在聊天页上传文件，或先把文件放到工作空间。
2. 在聊天里说明要处理哪个文件。
3. 说清楚你想要的结果格式。

可以这样说：

```text
请阅读我刚上传的 PDF，提炼 5 条核心结论，并列出需要人工复核的数据。
```

```text
根据这个 Excel，按客户类型汇总本月销售变化，输出一张 Markdown 表格。
```

注意：

- 大文件可能需要更长处理时间。
- 涉及合同、财务、投资、合规内容时，AI 输出应作为草稿或辅助分析，最终需要人工复核。

### 场景三：生成汇报、方案和研究材料

适合用来生成 PPT 大纲、研究报告、拜访提纲、竞品分析、管理层简报。

可以这样说：

```text
帮我准备一份拜访银行零售业务部门的交流提纲，主题是 AI 坐席助手和客户经营提效。
```

```text
围绕“AI Agent 在企业办公中的落地”，生成一份 6 页 PPT 大纲，每页给标题和 3 个要点。
```

如果已经安装了相关技能，也可以直接要求产物：

```text
使用 PPT 技能生成一份汇报材料，主题是智能体在银行运营提效中的应用场景。
```

使用建议：

- 先让智能体出“大纲”，确认方向后再生成完整材料。
- 需要引用来源时，要求它列出资料来源和不确定点。
- 对外材料要明确“不要编造数据，没有依据的地方标注待补充”。

### 场景四：安装和使用技能

技能是给智能体增加专门能力的模块，例如生成 PPT、处理 Word、做网页检索、分析特定业务数据。

使用方式：

1. 打开“技能”或“技能广场”。
2. 找到需要的技能并安装。
3. 回到聊天页，用自然语言说明要用这个技能做什么。

可以这样说：

```text
用已安装的 PPT 技能，把这份调研提纲做成 8 页演示文稿。
```

```text
根据这个 Word 模板，帮我生成一版客户拜访纪要。
```

如果你不知道需要哪个技能，可以直接问：

```text
我想把一段研究内容做成 PPT，应该安装或使用哪个技能？
```

### 场景五：创建定时任务

适合定时提醒、日报周报、天气提醒、定期检索、周期性总结。

可以在“定时任务”页面手动创建，也可以在聊天里直接说：

```text
每天早上 8 点查一下北京天气，简要总结后发到微信。
```

```text
每周五 17:30 提醒我整理本周项目进展，并给我一个周报模板。
```

```text
明天上午 9:17 执行一次，回复“测试任务 ping”。
```

说明：

- 定时任务绑定到当前员工智能体。
- 子智能体只能看到自己的任务，不会天然读取母智能体或其他智能体的任务。
- 投递到微信、飞书前，需要先完成对应渠道绑定。

### 场景六：绑定微信或飞书

频道用于让智能体从外部消息工具里收发消息。

微信常见用法：

1. 打开“频道”页面。
2. 选择微信绑定。
3. 按页面提示完成扫码或确认。
4. 在微信里给智能体发一条消息测试。

适合：

- 在微信里直接问智能体问题。
- 接收定时任务结果。
- 接收重要任务提醒。

如果绑定后收不到消息：

- 先在微信里主动给智能体发一句话，激活会话。
- 回到“频道”页面重新测试发送。
- 如果换环境重新绑定，旧环境可能仍显示历史状态，但实际投递以当前有效会话为准。

### 场景七：管理记忆

记忆用于保存长期偏好、个人背景、常用格式、业务上下文。

适合记录：

- 你的岗位、部门、常用工作场景。
- 常用汇报格式、语气偏好、输出模板。
- 常见客户、项目、产品简称。

可以在“记忆”页面编辑，也可以在聊天里说：

```text
记住：以后给我写周报时，按“本周进展、风险问题、下周计划”三段输出。
```

建议：

- 只保存长期稳定的信息。
- 不要把密码、密钥、银行卡号等敏感信息写入记忆。

### 场景八：组织协作

协作用于把一个任务分发给组织成员或其他智能体，再把结果汇总回来。

适合：

- 收集团队周报。
- 向多人收集项目进展。
- 让不同角色分别完成调研、分析、写作。
- 发起跨部门信息收集。

基本流程：

1. 发起人创建协作任务。
2. 选择参与成员或智能体。
3. 参与者在自己的工作台处理任务。
4. 结果回传后，发起人查看汇总。

可以这样描述任务：

```text
请每位成员按“本周进展、阻塞风险、下周计划”提交项目周报，最后汇总成一份团队周报。
```

### 场景九：使用工作空间

工作空间是当前智能体的文件区，适合保存输入文件和生成结果。

你可以：

- 上传资料给智能体读取。
- 下载智能体生成的文件。
- 删除不需要的临时文件。
- 在聊天中引用工作空间里的文件继续处理。

使用建议：

- 文件名尽量清楚，例如 `2026Q1客户反馈汇总.xlsx`。
- 生成的重要文件及时下载或归档。
- 不要把无关敏感文件长期放在工作空间。

## 3. 怎么把任务说清楚

一个好指令通常包含四件事：

| 要素 | 示例 |
|---|---|
| 背景 | “我要给银行客户做一次售前交流” |
| 目标 | “准备一份交流提纲” |
| 约束 | “不要超过 1 页，语气正式” |
| 格式 | “用表格输出，列为主题、问题、预期价值” |

推荐模板：

```text
背景：我要……
目标：请帮我……
要求：语气……，长度……，不要……
格式：请输出为……
```

示例：

```text
背景：我要拜访一家城商行的运营管理部。
目标：请帮我准备 AI 员工智能体的交流提纲。
要求：偏业务价值，不要堆技术名词。
格式：输出为表格，列包括交流主题、客户痛点、可展示能力、预期收益。
```

## 4. 管理员常用配置

管理员可以在后台处理这些事情：

- 查看和管理用户。
- 配置员工智能体的开放范围。
- 配置品牌名称、Logo、页面标题和默认 system prompt。
- 配置 SMTP 邮件。
- 查看使用统计和审计信息。
- 管理首页场景和体验入口。

### 开放申请范围

如果希望所有注册用户都能申请员工智能体，需要把智能体可见性配置为公开。生产环境调整前建议先确认资源容量和 OpenClaw 运行状态。

### 品牌配置

品牌配置会影响：

- 首页标题和浏览器标题。
- `/api/brand` 返回的品牌字段。
- 登录、邮件、管理后台里的默认品牌口径。
- 默认 system prompt 中的平台身份描述。

如果页面仍显示旧文案，通常是以下原因：

1. 前端没有重新 build。
2. PM2 没有重启。
3. 数据库 `system_configs` 里的 `brand_*` 覆盖值仍是旧配置。
4. 浏览器缓存未刷新。

## 5. 部署和升级

### 一键部署

在 Ubuntu 服务器上执行：

```bash
curl -fsSL https://raw.githubusercontent.com/bitlhk/employee-agent/main/scripts/bootstrap-install.sh | bash
```

脚本会自动安装基础依赖、Node.js、pnpm、PM2，拉取代码，初始化 `.env` 和数据库，构建前端并启动服务。

### 指定安装参数

```bash
curl -fsSL -o /tmp/bootstrap-install.sh \
  https://raw.githubusercontent.com/bitlhk/employee-agent/main/scripts/bootstrap-install.sh

bash /tmp/bootstrap-install.sh \
  --dir "$HOME/employee-agent" \
  --port 5180 \
  --host 111.119.236.165 \
  --branch main
```

常用参数：

| 参数 | 说明 |
|---|---|
| `--dir` | 安装目录，默认 `$HOME/employee-agent` |
| `--port` | 服务端口，默认 `5180` |
| `--host` | 访问地址使用的公网 IP 或域名 |
| `--branch` | Git 分支，默认 `main` |
| `--repo` | Git 仓库地址 |
| `--skip-start` | 只初始化，不启动 |
| `--skip-mysql` | 不安装本机 MySQL |
| `--db-mode existing` | 使用已有数据库 |
| `--overwrite-env` | 重新生成 `.env` |
| `--dry-run` | 只打印动作，不实际执行 |

### 初始化管理员

```bash
cd ~/employee-agent
corepack pnpm tsx scripts/init-admin.ts \
  --email=admin@example.com \
  --password='换成强密码' \
  --name='Admin'
```

### 升级

重复执行一键脚本即可升级已有安装：

```bash
curl -fsSL https://raw.githubusercontent.com/bitlhk/employee-agent/main/scripts/bootstrap-install.sh | bash
```

默认不会覆盖已有 `.env`。如需重建 `.env`：

```bash
curl -fsSL https://raw.githubusercontent.com/bitlhk/employee-agent/main/scripts/bootstrap-install.sh | bash -s -- --overwrite-env
```

## 6. 运维排查

### 检查服务

```bash
pm2 status employee-agent
pm2 logs employee-agent
pm2 restart employee-agent
```

如果是旧安装目录或兼容升级环境，PM2 进程名可能仍是 `linggan-claw`，以实际 `pm2 list` 为准。

### 检查 OpenClaw

```bash
cd ~/employee-agent
bash scripts/check-local-openclaw-node.sh
```

重点确认：

- OpenClaw Gateway 已启动。
- `CLAW_GATEWAY_URL` 指向正确 Gateway。
- `CLAW_GATEWAY_TOKEN` 与 OpenClaw 配置一致。
- `FRONTEND_URL` 与浏览器访问地址一致。
- WebSocket / SSE 可以正常访问。

### 重新构建和重启

```bash
cd ~/employee-agent
corepack pnpm check
corepack pnpm build
pm2 restart employee-agent
```

### 常见问题

**申请员工智能体失败**

先检查 OpenClaw Gateway、数据库连接和 PM2 日志。

**聊天一直没有响应**

优先检查 OpenClaw，再看 PM2 日志。

**微信或飞书收不到消息**

进入“频道”页面重新测试绑定状态。微信长时间未互动时，先在微信里主动给智能体发一句话。

**首页或登录页仍是旧文案**

重新 build 前端并重启服务，同时检查数据库 `brand_*` 配置是否覆盖了代码默认值。

**多窗口会串消息吗**

不会。Web、微信、飞书等渠道使用独立会话标识，不同浏览器窗口也会分配不同 web conversation。
