# 4xx 错误自动封禁功能实现说明

## 功能概述

实现了基于 4xx 错误数量的自动 IP 封禁功能。当某个 IP 在短时间内产生过多的 4xx 错误时，系统会自动将该 IP 添加到黑名单中，从而阻止其继续访问。

## 主要功能

1. **4xx 错误追踪**：实时追踪每个 IP 的 4xx 错误数量
2. **自动封禁**：当错误数量超过阈值时，自动将 IP 添加到黑名单
3. **可配置阈值**：支持通过系统配置表自定义自动封禁阈值
4. **防重复封禁**：避免重复将同一个 IP 添加到黑名单
5. **自动解封**：封禁的 IP 默认在 24 小时后自动解封（可配置）

## 工作原理

### 错误追踪机制

- **时间窗口**：15 分钟滑动窗口
- **错误计数**：统计每个 IP 在 15 分钟内产生的 4xx 错误数量
- **排除规则**：静态资源（如 `/assets/`, `/images/` 等）的 404 错误不计入统计

### 自动封禁触发条件

当满足以下条件时，IP 会被自动封禁：

1. **错误数量阈值**：15 分钟内产生的 4xx 错误数量 >= 自动封禁阈值（默认 30 个）
2. **不在黑名单中**：IP 当前不在黑名单中
3. **未自动封禁过**：该 IP 在当前错误记录周期内尚未被自动封禁

### 封禁行为

- **封禁类型**：`blacklist`（黑名单）
- **严重程度**：`high`（高）
- **过期时间**：默认 24 小时后自动解封
- **记录信息**：包含错误数量、错误率、最近 10 个错误的详细信息

## 配置说明

### 系统配置项

可以通过系统配置表设置自动封禁阈值：

- **配置键**：`auto_block_4xx_threshold`
- **配置值**：数字类型，表示触发自动封禁的 4xx 错误数量阈值
- **默认值**：30（如果未配置或配置无效，使用此默认值）

### 配置示例

```typescript
// 通过 tRPC API 设置自动封禁阈值为 50
await trpc.systemConfigs.upsert.mutate({
  key: "auto_block_4xx_threshold",
  value: "50",
  description: "自动封禁 4xx 错误阈值：15 分钟内超过此数量的 4xx 错误将自动封禁 IP"
});
```

## 使用说明

### 1. 默认行为

系统默认启用自动封禁功能（生产环境）：
- 默认阈值：30 个 4xx 错误（15 分钟内）
- 封禁时长：24 小时
- 开发环境：只记录日志，不实际封禁

### 2. 自定义配置

管理员可以通过后台管理系统或 API 调整自动封禁阈值：

```typescript
// 设置更严格的阈值（20 个错误即封禁）
await trpc.systemConfigs.upsert.mutate({
  key: "auto_block_4xx_threshold",
  value: "20",
  description: "自动封禁 4xx 错误阈值"
});

// 设置更宽松的阈值（50 个错误才封禁）
await trpc.systemConfigs.upsert.mutate({
  key: "auto_block_4xx_threshold",
  value: "50",
  description: "自动封禁 4xx 错误阈值"
});
```

### 3. 查看封禁记录

管理员可以通过以下方式查看自动封禁的 IP：

1. **IP 管理页面**：查看 `ip_management` 表中 `type = 'blacklist'` 的记录
2. **安全日志**：查看 `security_logs` 表中 `reason` 包含"自动加入黑名单"的记录
3. **IP 管理记录**：查看 `notes` 字段中包含 `autoBlocked: true` 的记录

### 4. 手动解封

如果误封了正常用户，管理员可以手动解封：

```typescript
// 通过 tRPC API 删除或禁用 IP 管理记录
await trpc.ipManagement.delete.mutate({ id: ipManagementId });
// 或
await trpc.ipManagement.update.mutate({ 
  id: ipManagementId, 
  updates: { isActive: "no" } 
});
```

## 注意事项

1. **开发环境**：开发模式下不会实际封禁 IP，只会记录日志
2. **静态资源**：静态资源的 404 错误不计入错误统计，避免误封
3. **重复封禁**：系统会避免重复将同一个 IP 添加到黑名单（在同一错误记录周期内）
4. **过期时间**：封禁的 IP 默认 24 小时后自动解封，可以通过修改 IP 管理记录调整
5. **性能考虑**：错误追踪使用内存存储，生产环境建议使用 Redis 等持久化存储

## 监控和告警

### 安全日志

每次自动封禁都会记录到安全日志：
- **严重程度**：`critical`
- **原因**：`IP 已自动加入黑名单：15分钟内产生 X 个 4xx 错误`
- **详细信息**：包含错误数量、错误率、阈值、最近错误列表

### 控制台日志

自动封禁时会输出警告日志：
```
[ErrorTracking] IP 192.168.1.1 已自动加入黑名单：15分钟内产生 35 个 4xx 错误
```

## 后续优化建议

1. **持久化存储**：将错误追踪数据从内存迁移到 Redis，支持多实例部署
2. **更灵活的配置**：支持配置封禁时长、错误窗口时间等参数
3. **白名单机制**：支持配置 IP 白名单，白名单中的 IP 不会被自动封禁
4. **分级封禁**：根据错误数量设置不同的封禁时长（错误越多，封禁时间越长）
5. **自动解封通知**：封禁到期前通知管理员，便于评估是否需要延长封禁
