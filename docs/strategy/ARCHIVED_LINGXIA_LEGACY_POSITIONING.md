# 灵虾 (Lingxia) 系统架构总览

**版本**: 1.0 · 面向领导/客户层汇报版
**日期**: 2026-05-05
**性质**: 产品架构 + 价值定位

---

## 一句话定位

灵虾是**面向银行/金融行业的 AI 任务交付平台**——把 AI 大模型 + 智能体 + 工具链整合为可审计、可追溯、可交付的工作流，让金融机构无需理解技术细节即可让 AI 完成实际任务。

这是最容易被混淆的部分，单独画清楚：

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │

│   "我要一份新能源车险尽调报告"                                     │
│        │                                                           │
│        ▼                                                           │
│   ┌──────────────────────────────────────────────┐                 │
│   │  灵虾  (Lingxia)                              │                 │
│   │  ─────                                       │                 │
│   │  • 提供"任务"的产品语义                       │                 │
│   │  • 编排多个 AI 智能体协作                     │                 │
│   │  • 管理用户身份/权限/合规/审计                │                 │
│   │  • 多租户隔离 (Space-scoped)                  │                 │
│   │  • 渠道分发 (微信/飞书)                       │                 │
│   │  • 产物渲染 (PPT/Word/Excel)                  │                 │
│   └────────────┬──────────────────────┬──────────┘                 │
│                │                      │                            │
│                │ "调用主聊天"          │ "调用专业智能体"           │
│                ▼                      ▼                            │
│   ┌─────────────────────┐    ┌───────────────────────────┐         │
│   │   OpenClaw          │    │   Hermes                   │         │
│   │   ────────          │    │   ──────                   │         │
│   │   通用对话 + 工具   │    │   金融领域专业智能体集群    │         │
│   │   执行运行时        │    │   (信贷/债券/保险/财富)     │         │
│   │                    │    │                            │         │
│   │   • 主聊天 Agent    │    │   • task-credit-risk        │         │
│   │   • 工具编排        │    │   • task-bond               │         │
│   │   • Skill 加载      │    │   • task-claim-ev           │         │
│   │   • 工作区隔离      │    │   • task-my-wealth          │         │
│   │   • 多模型 Adapter  │    │   • task-hermes             │         │
│   └─────────────────────┘    └───────────────────────────┘         │
│   位置: 灵虾本机部署          位置: AWS 海外 (经 SSH 隧道接入)       │
│                                                                    │
│   [由 Anthropic 生态衍生]      [灵虾内部基于 Hermes 框架]            │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

**简单说**：

- **OpenClaw** = 灵虾**主聊天底座**，相当于"通用对话引擎 + 工具执行平台"
- **Hermes** = 灵虾**金融专业智能体平台**，每个智能体是一个领域专家（信贷分析师/债券研究员等）
- **灵虾** = **产品层**，把这两套底座**包装成银行能用的任务**，加上身份/权限/合规/审计/渠道分发

银行客户**永远只看到灵虾**。OpenClaw 和 Hermes 是底座，不直接面向客户。

## 三、用户提交一个任务的全链路

以"AI 趋势洞察 PPT"为例：

```
┌────────────────────────────────────────────────────────────────────┐
│ Step 1  用户在灵虾任务工作台选"AI 趋势洞察 PPT"                    │
│         填入需求:"红杉 AI Ascent 2026 大会观点提炼"                │
│                                                                    │
│ Step 2  灵虾 TaskTemplateRunner 加载任务模板                        │
│         → 解析: 需要 3 个 AI 专员协作                              │
│         → 校验: 用户权限/可见性/合规边界                            │
│                                                                    │
│ Step 3  闻舟 (AI) · 趋势洞察师 [Hermes 智能体]                      │
│         → 检索全网资料 (Bocha + Tavily + Brave)                    │
│         → 分级筛选来源 (官方/媒体/评论/低质)                        │
│         → 给每条来源稳定 ID (src_001, src_007...)                  │
│         产出: 16 条候选证据 + 引用编号                              │
│                                                                    │
│ Step 4  墨衡 (AI) · 研究审阅员 [DeepSeek 模型]                      │
│         → 读闻舟证据包                                              │
│         → 提炼 5 个核心论点                                         │
│         → 每个论点必须带 [src_NNN] 引用                             │
│         产出: 论点结构 + 引用链                                     │
│                                                                    │
│ Step 5  简页 (AI) · PPT 制作师 [Claude Code 智能体]                 │
│         → 接收墨衡论点 + 闻舟引用                                   │
│         → 选 layout (封面/对比/矩阵/引用页)                         │
│         → 应用主题 (8 套视觉风格之一)                               │
│         → 渲染产出 .pptx + .html 预览                               │
│                                                                    │
│ Step 6  灵虾合规层自动追加                                          │
│         → "AI 生成"标识                                             │
│         → 资料来源页 (从 src_NNN 列表生成)                          │
│         → 任务级免责声明 (按任务类型: 投顾/代码/事实)               │
│         → 审计日志 (谁/何时/任务版本/产物哈希)                      │
│                                                                    │
│ Step 7  用户在工作台看到:                                           │
│         → 完整 PPT 预览                                             │
│         → 下载 .pptx                                                │
│         → 保存到我的工作区 (副本，不动原产物)                       │
│         → 每条结论可点回引用来源                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 六、银行场景核心价值（差异化卖点）

```
┌────────────────────────────────────────────────────────────────────┐
│                                                                    │
│  灵虾 vs 通用 AI 助手 (ChatGPT / 文心 / 豆包) 的差异                │
│                                                                    │
│  ┌────────────────┐  ┌─────────────────────┐                       │
│  │  通用 AI 助手  │  │  灵虾                │                       │
│  └────────────────┘  └─────────────────────┘                       │
│  • 单 agent 对话    │ • 多专员协作完成任务                          │
│  • 无引用来源       │ • 每条结论可追溯到原始来源                    │
│  • 数据进公网       │ • 私有化 / 数据不出客户域                     │
│  • 输出无审计       │ • 7 年审计日志 (banking 默认)                 │
│  • 无身份/权限      │ • 多租户 + 空间级隔离                         │
│  • 无合规边界       │ • 投顾/代码/研究分级免责声明                  │
│  • 不集成办公       │ • 微信/企微/飞书原生集成                      │
│  • 无定时调度       │ • Cron 调度 + 渠道推送                        │
│  • 无产物管理       │ • PPT/Word/Excel 完整预览 + 导出              │
│  • 无任务模板       │ • 预置任务 + 用户自定义任务链                 │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

**核心论点**：

> 银行客户买的不是"接入大模型"，是"**让 AI 在合规边界内完成可交付的工作**"。
> 灵虾的产品层是这个差异化的载体。

## 十、给领导的一页结论

> **灵虾不是"接入大模型的网关"，是"AI 任务交付平台"**。
>
> 1. **底座可换**：今天用 Anthropic Claude，明天可换国产大模型，客户感知不变
> 2. **能力可扩**：每加一个智能体不重写 UI，每加一个任务不重写后端
> 3. **合规天然**：审计/引用/隔离/免责从架构层做到，不是事后补丁
> 4. **客户可控**：私有化部署、数据不出客户域、多租户硬隔离
> 5. **场景对路**：银行实际用得上的是"完成任务"不是"对话聊天"
>
> 这 5 条加起来，是"通用 AI 助手"做不到的事。也是灵虾在金融行业的位置。
